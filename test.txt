--------------------------------------------------------------------------------
 4. THE JUDGE LOOP  (the heart of the system)
--------------------------------------------------------------------------------
A small poll helper — paste this function into your shell:
$ verdict(){ for i in $(seq 1 40); do
    V=$(curl -s $BASE/submissions/$1 -H "Authorization: Bearer $2" \
        | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['verdict'])");
    [ "$V" != PENDING ] && [ "$V" != JUDGING ] && { echo "$V"; return; }; sleep 2;
  done; echo TIMEOUT; }

NOTE: the FIRST submission in a new language may take ~20s extra the very first
time (image warm-up). After that it's fast.

(a) CORRECT solution -> AC
$ SID=$(curl -s -X POST $BASE/submissions -H "Authorization: Bearer $BOB" \
    -H 'Content-Type: application/json' \
    -d '{"problemId":'$PID',"language":"PYTHON","sourceCode":"a,b=map(int,input().split())\nprint(a+b)"}' \
    | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['id'])")
$ verdict $SID $BOB
✅ AC

(b) WRONG answer -> WA   (wait ~11s between submits: rate limit is 1/10s)
$ sleep 11
$ SID=$(curl -s -X POST $BASE/submissions -H "Authorization: Bearer $BOB" \
    -H 'Content-Type: application/json' \
    -d '{"problemId":'$PID',"language":"PYTHON","sourceCode":"a,b=map(int,input().split())\nprint(a-b)"}' \
    | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['id'])")
$ verdict $SID $BOB
✅ WA

(c) COMPILE error -> CE  (use a broken C++ program)
$ sleep 11
$ SID=$(curl -s -X POST $BASE/submissions -H "Authorization: Bearer $BOB" \
    -H 'Content-Type: application/json' \
    -d '{"problemId":'$PID',"language":"CPP","sourceCode":"int main(){ broken"}' \
    | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['id'])")
$ verdict $SID $BOB
✅ CE

(d) TIME LIMIT -> TLE  (infinite loop)
$ sleep 11
$ SID=$(curl -s -X POST $BASE/submissions -H "Authorization: Bearer $BOB" \
    -H 'Content-Type: application/json' \
    -d '{"problemId":'$PID',"language":"PYTHON","sourceCode":"while True: pass"}' \
    | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['id'])")
$ verdict $SID $BOB
✅ TLE

(e) MEMORY LIMIT -> MLE  (allocate way past 256MB)
$ sleep 11
$ SID=$(curl -s -X POST $BASE/submissions -H "Authorization: Bearer $BOB" \
    -H 'Content-Type: application/json' \
    -d '{"problemId":'$PID',"language":"PYTHON","sourceCode":"x=bytearray(600*1024*1024)\nprint(len(x))"}' \
    | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['id'])")
$ verdict $SID $BOB
✅ MLE

(f) RUNTIME error -> RE
$ sleep 11
$ SID=$(curl -s -X POST $BASE/submissions -H "Authorization: Bearer $BOB" \
    -H 'Content-Type: application/json' \
    -d '{"problemId":'$PID',"language":"PYTHON","sourceCode":"print(1//0)"}' \
    | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['id'])")
$ verdict $SID $BOB
✅ RE

 11. RESILIENCE  (crash + restart safety)
--------------------------------------------------------------------------------
Worker down -> submission waits in the durable queue -> judged on restart:
$ docker stop codearena-judge
$ sleep 11
$ SID=$(curl -s -X POST $BASE/submissions -H "Authorization: Bearer $BOB" \
    -H 'Content-Type: application/json' \
    -d '{"problemId":'$PID',"language":"PYTHON","sourceCode":"a,b=map(int,input().split())\nprint(a+b)"}' \
    | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['id'])")
$ curl -s $BASE/submissions/$SID -H "Authorization: Bearer $BOB" | grep -o '"verdict":"[A-Z]*"'
✅ "verdict":"PENDING"  (no worker to judge it yet)
$ docker exec codearena-rabbitmq rabbitmqctl list_queues name messages | grep judge.queue
✅ judge.queue   1     (message safely waiting)
$ docker start codearena-judge
$ verdict $SID $BOB
✅ AC  (message survived the worker being down — nothing lost)

Full cold restart keeps all data:
$ docker compose -f docker-compose.full.yml restart
$ until curl -sf http://localhost:8080/actuator/health | grep -q UP; do sleep 3; done; echo UP
$ curl -s "$BASE/problems" | python3 -c "import sys,json;print('problems:',json.load(sys.stdin)['data']['totalElements'])"
✅ problems still there
$ docker exec codearena-postgres psql -U codearena -d codearena -t \
    -c "SELECT count(*) FROM submissions WHERE verdict IN ('PENDING','JUDGING');"
✅ 0  (nothing stuck)
